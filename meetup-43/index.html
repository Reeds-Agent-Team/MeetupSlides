BIP 340
Schnorr
Signatures
Replaces ECDSA
Cryptographer Claus Schnorr's scheme — whose patent expired in 2008 — lets multiple parties combine their keys and signatures into one. A 3-of-5 multisig looks identical on-chain to a single-sig spend. Signatures shrink from ~72 bytes to 64 bytes, and batches of sigs can be verified together, cutting node validation work.
Key aggregation Batch verify 64-byte sigs
BIP 341
Taproot
/ MAST
Merkelized Alternative Script Trees
Outputs look like a single key. If everyone agrees, spend with one Schnorr sig — no script ever hits the chain. If a dispute arises, reveal only the one branch of the Merkle tree you actually use. Lightning channel opens/closes and complex multisig vaults become indistinguishable from ordinary payments.

BIP 342
Tapscript
Updated scripting language
Upgrades Bitcoin Script to read Schnorr sigs and unlocks the full MAST feature set. Removes the old 10,000-byte script size cap. Adds an upgrade path — new opcodes can be added in future soft forks without a new address type. Lays groundwork for PTLCs, vaults, DLCs, and BitVM-style constructions.
No size limit PTLC-ready Future opcodes

How Taproot Works + The Activation Fight
How a Taproot output is spent
The Activation Fight (2020–2021)
LOT = false
If 90% of miners don't signal within the window, the upgrade simply expires. Miners effectively hold a soft veto — echoing the 2017 SegWit standoff that nearly ended in a chain split.
LOT = true (UASF)
Nodes eventually start rejecting non-signaling blocks, forcing activation regardless of miners. More confrontational — but defenders argued it was the only way to guarantee users, not miners, control protocol upgrades.
⚡ Speedy Trial — The Compromise
Proposed by Russell O'Connor to break the deadlock: a tight 3-month window for miners to signal. If 90% locked in, Taproot activates automatically 5 months later. If not — everyone learns something and a UASF is likely next.

Miners locked in at block 687,285 on Jun 12, 2021. 2,016 blocks · 99.7% signaled ₿
Unlike SegWit's blocksize civil war, virtually no one opposed Taproot itself — the fight was entirely about who should have the final say over activation: miners or users.
